# Innocence 桌面端 UI 实现约束

## 1. 登录页输入框约束

- Windows 桌面端登录页曾出现大面积灰色输入区域。
- 现象表现：
  - 灰色区域与输入框位置完全重合
  - 鼠标移入后会变成文本输入状态
  - 灰色区域会随输入框尺寸变化而变化
- 结论：
  - 这不是额外遮罩层问题
  - 主要是 `TextField + InputDecoration + 填充背景/自定义容器` 的组合在当前桌面端壳层下出现异常绘制

### 已验证可用方案

- 登录页输入框改用更底层的 `EditableText`
- 输入框外观由外层 `Container / AnimatedContainer` 单独负责
- 占位提示文本单独绘制，不依赖 `InputDecoration` 的默认 hint/fill 体系
- 输入区域保持：
  - 单行
  - 固定高度
  - 无额外 filled 背景层
  - 无复杂玻璃叠层

### 后续约束

- 桌面端登录页不要直接恢复成旧的 `TextField + InputDecoration(filled: true)` 方案
- 如果后续必须恢复 Material 输入框样式，先单独做 Windows 实机验证，再合并到主界面
- 桌面端登录页优先保证稳定显示，再谈高级视觉效果

## 1. 修改登录页时的检查清单

- 标题是否真正居中
- 关闭按钮是否过大
- 标题区域是否可以拖动窗口
- 鼠标移入输入区时，是否只出现正常文本光标
- 输入框是否仍有灰色块、灰色底、额外遮挡条
- Windows 实机截图是否与设计预期一致
